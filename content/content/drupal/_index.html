Routes

Drupal 8's routing system is heavily based on Symfony's. Drupal's routing system can do everything Symfony's can (and more), and both use the same syntax to define routes.

Routing system overview | Routing system | Drupal Wiki guide on Drupal.org Routing (Symfony 4.4 Docs)

Example

example.name:
  path: '/example/{name}'
  defaults:
    _controller: '\Drupal\example\Controller\ExampleController::content'
  requirements:
    _permission: 'access content'

More info on params: Using parameters in routes | Parameters in routes | Drupal Wiki guide on Drup…

General

These are just some notes to keep in mind when trying to debug an issue in Drupal. Some issues I've run into time and time again, that are helpful to run thru as a process of elimination.

  1. Is it cached?

    Meaning, is the page cached? Make sure to check here because if you are logged in as the admin, you are skipping the cache. Make sure to check as an anon user.

  2. Check the path

    Check to see what's going on with the URL at all times. There is a lot of telling information in the URL, as this is the entry point for all things web/Drupal.

    For instance, I ran into an issue where I was trying to figure out what was going on with a groupmenu item, thinking that it had something to do with a custom link that was inside of the groupmenu parent, when in fact, it had to do with a path exclusion that was written for the page title block.

    The page title was excluded for everything past the path /people/*, so if something was inserted into that part of the menu, then the h1 would not be rendered for that page.

PHP

Interfaces

Interface methods are to all be made public. There are no private methods in an interface.

From php.net:

https://www.php.net/manual/en/language.oop5.interfaces.php Object interfaces allow you to create code which specifies which methods a class must implement, without having to define how these methods are implemented.

This is a basic example:

interface Something
{
    public function setVariable($name, $var);
    public function getHtml($template);
}

This interface could then be extended by a class.

class Template implements Something {
    private $vars = [];

    public function setVariable($name, $var) {
        $this->vars[$name] = $var;
    }
}

Preprocess theme variables for a specific theme hook.

This hook allows modules to preprocess theme variables for a specific theme hook. It should only be used if a module needs to override or add to the theme preprocessing for a theme hook it didn't define.

Webforms

Tracking query params via hidden field

The Token module will allow you to use the following: [current-page:query:?]

Replace "?" with the parameter you want to get values from. E.g., "?source=" would be [current-page:query:source]

Add a hidden field on your webform, and place this as its default value.

Cache

Cache tags

Example that Natalie gave to me to get the block to stop caching. Should review this more and add some better notes on why and how this actually works.

use Drupal\Core\Cache\Cache;
class RelatedContentConditionalBlock extends BlockBase {
    public function getCacheTags() {
    //With this when your node change your block will rebuild
    if ($node = \Drupal::routeMatch()->getParameter('node')) {
        //if there is node add its cachetag
        return Cache::mergeTags(parent::getCacheTags(), array('node:' . $node->id()));
    } else {
        //Return default tags instead.
        return parent::getCacheTags();
    }
    }

    public function getCacheContexts() {
    //if you depends on \Drupal::routeMatch()
    //you must set context of this block with 'route' context tag.
    //Every new route this block will rebuild
    return Cache::mergeContexts(parent::getCacheContexts(), array('route'));
    }
}

Date range

In the event you need to use date range field and split it up.

Also reference for: template_preprocess_field()

/*
 * Implements template_preprocess_field().
*/
function sgcuikit_preprocess_field(&$variables, $hook) {
    if ($variables['element']['#field_name'] == 'field_event_date') {
    $content = $variables['items'][0]['content'];
    $start_date = $content['start_date'];
    $end_date = $content['end_date'];

    // Example: SEPTEMBER 25-27, 2022
    $start_date = date('F d', strtotime($start_date['#markup']));
    $end_date = date('d, Y', strtotime($end_date['#markup']));

    $processed_date = $start_date . '-' . $end_date;

    $variables['custom_date'] = $processed_date;
    }
}
